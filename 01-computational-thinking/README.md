## ⚙️ `[내 동작 분석]`

컴퓨팅 사고 4요소로 **「주간 운영 리포트 작성」** 업무를 해부합니다.

### 1️⃣ 분해 (Decomposition)

큰 업무를 **3단계 → 세부 작업**으로 쪼갭니다.

```
주간 운영 리포트 작성
├── A. 데이터 수집        ▸ 지표 추출 · 전주 대비 증감 계산 · VOC 로그 수집
├── B. 분석 · 해석        ▸ 이상치 탐지 · VOC 분류 · 원인 가설 · 핵심 이슈 Top 3
└── C. 문서화 · 공유      ▸ 템플릿 작성 · 요약 · 액션아이템 정리 · 슬랙/메일 공유
```

| 단계 | 세부 작업 | 소요 | 성격 |
|------|-----------|:----:|------|
| **A. 수집** | 지표 추출 / 증감 계산 / VOC 수집 | ~40분 | 단순 반복 |
| **B. 분석** | 이상치·VOC 분류 / 원인 / Top 3 선정 | ~60분 | 판단 필요 |
| **C. 문서화** | 템플릿·요약·액션아이템·공유 | ~50분 | 반복+작성 |

### 2️⃣ 패턴 (Pattern Recognition)

매주 똑같이 되풀이되는 **규칙**을 찾습니다.

- 🔁 **완전 반복(규칙 고정):** 지표 추출 · 증감 계산 · 템플릿 채우기 → *매주 동일*
- 🔁 **반복+판단:** VOC 분류 · 이상치 탐지 → *절차는 같고 해석만 다름*
- ⚠️ **병목/실수:** 수기 복붙에서 숫자 오기입, 과거 리포트를 매번 다시 찾는 비효율

> **인사이트:** "규칙 고정" 구간은 **자동화 1순위**, "반복+판단" 구간은 **AI 초안 보조** 대상.

### 3️⃣ 추상화 (Abstraction)

곁가지를 걷어내면, 이 업무는 결국 **정해진 입력 → 규칙 → 출력** 을 채우는 함수입니다.

```yaml
입력(Input):
  metrics:   지표 값 + 전주 값
  voc_logs:  고객 문의 텍스트 목록
규칙(Rule):
  증감률 = (이번주 - 전주) / 전주
  VOC → 사전 정의 카테고리로 분류
  |증감률| ≥ 임계치 → "주목 이슈"로 승격
출력(Output):
  summary:    3줄 요약
  highlights: 핵심 이슈 Top 3
  actions:    액션 아이템(담당자 포함)
```

### 4️⃣ 알고리즘 (Algorithm Design)

누구나 따라 할 절차로 정리하고, **자동화 수준**을 태깅합니다.
`🟢 자동` · `🟡 AI 보조(사람 검수)` · `🔴 사람 판단`

```text
function 주간운영리포트():
  1. metrics   ← 대시보드에서 지표 추출        🟢
  2. 증감률     ← 각 지표 전주 대비 계산         🟢
  3. voc       ← 문의 로그 수집                 🟢
  4. voc_tagged← 카테고리 분류(voc)             🟡
  5. issues    ← 급변 지표 + VOC 급증 항목       🟡
  6. top3      ← issues 중 중요도 상위 3개       🔴
  7. cause     ← 이슈별 원인 가설 작성          🔴
  8. draft     ← 템플릿에 값·요약 채우기        🟡
  9. 사람 검수 후 슬랙/메일 공유                🔴
```

| 자동화 수준 | 단계 | 비중 |
|-------------|------|:----:|
| 🟢 자동 | 1·2·3 | ~35% |
| 🟡 AI 보조 | 4·5·8 | ~40% |
| 🔴 사람 판단 | 6·7·9 | ~25% |

➡️ **약 75%를 AI로 자동화/초안화 가능** → 이 지점이 곧 내 개인 프로젝트가 된다.

## 📤 `[출력 요구]`

> **표 · 트리 · 의사코드를 포함한 마크다운 리포트 형식**으로, **한국어로.**

- 각 지표는 **수치 + 전주 대비 증감률**을 함께
- 서두에 **3줄 요약**, 본문에 **핵심 이슈 Top 3**, 말미에 **액션 아이템(담당자 포함)**
- 자동화 가능 단계는 🟢🟡🔴 로 표시해 **사람이 검수할 지점**이 한눈에 보이게

## ✅ `[검증 기준]`

> **실제로 매주 운영 리포트를 작성해 본 기획자 / PM** 한 사람이,
> **"이 분해대로 따라 하면 작성 시간을 절반으로 줄일 수 있고, 어디를 자동화할지 명확하다"** 라고 평가할 것.

**세부 성공 지표**

| 기준 | 목표 |
|------|------|
| ⏱️ 리포트 작성 시간 | 150분 → **60분 이하** (약 60% 절감) |
| 🎯 수치 오기입 | **0건** |
| 📝 초안 활용률 | 생성된 초안을 그대로/약간 수정해 사용한 비율 **80% 이상** |

<br/>

---

- **문제:** *기획/운영자가 매주 1~2시간 쓰는 반복적 운영 리포트 작성을, 수집–분석–초안까지 대신 처리하는 AI 에이전트가 없다.*
- **해결 아이디어:** 지표 조회 · VOC 분류 · 문서 생성 **도구(Tool)** 를 갖춘 **「주간 운영 리포트 에이전트」** — 사람은 Top 3·원인·최종 검수만 수행
- **다음 미션 연결:** 프롬프트 엔지니어링(요약 품질) → Tool Calling/에이전트(도구 연동) → 파이널 프로젝트(실제 구현·배포)
